# SmartFirm Grow Kit - Project Rules

## Content Rules
All content rules and writing guidelines are in `CONTENT_PRINCIPLES.md`. Read it before writing or editing any page content.

Key rules:
- NEVER use em-dashes in any content
- NEVER use "guaranteed" or guarantee language
- Use consultative, conditional language ("can include", "typically", "often")
- Focus on outcomes over feature lists
- No specific time commitments, metrics, or deliverable lists unless from real case studies
